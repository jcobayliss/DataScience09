Prediction of the cost of solving CSPs by using different heuristics

A very naive and not very accurate approach

José Carlos Ortiz Bayliss

Constraint Satisfaction Problems (CSPs)

Constraint satisfaction problems consist of:

  • A set of variables.
  • A set of available values for each variable.
  • A set of constraints that restrict the values such variables can simultaneusly take.

The goal when solving CSPs is either to find an assignment to all the variables that satisfies all the constraints or to prove that the instance is unsatisfiable.

Heuristics

When solving CSPs, the variables are assigned a value, one at the time. To decide the order in which the variables are assigned we use heuristics.

A heuristic is a 'rule of thumb': we do not know exactly how and why they work, but somehow the experience tells as that they work well under certain circumstances.

In this small project we use three heuristics:

  • DOM. Considers first the variables with the smallest domains.
  • DEG. Tries first the variables highly connected to other variables.
  • K. Considers combinatorial aspects of the instances to order the variables.

It is difficult in practice to predict which heuristic is the best option.

Mapping from instances to suitable heuristics

There are some features that allow us to characterize the CSP instances. In this case we use only the constraint density and tightness (not really a good idea but it is simple enough for the purpose of this project). The Prediction Model

For each point in the space p1 x p2 we are interested in finding the best heuristic without having to run the three of them. We want to predict the cost of these heuristics to decide which one to use (the one with the less predicted cost).

We proposed a very naive prediction model:

library(reshape2)
 data <- read.csv(file = 'data.csv', header = TRUE, sep = ',')  
  checks <- data[data$SET == 'coloring', c(3, 4, 11, 13, 15)]
  names(checks) <- c('P1', 'P2', 'DOM', 'DEG', 'K')
  checks <- melt(data = checks, id.vars = c('P1', 'P2'), variable.name = 'HEURISTIC', value.name = 'COST')
  checks = checks[checks$HEURISTIC == heuristic, ]
  model <- lm(COST ~ P1 + P2, data = checks)

Please note that this model is proposed only for the purpose of this class and it is not intended for deployment on a real scenario, as both the number of instances solved and the features analyzed are insufficient to provide good results.
